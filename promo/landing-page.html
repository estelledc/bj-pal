Public release · v6.26.0 CASE 02 / PRODUCT SYSTEMS

BJ-Pal / 北京搭子

把“去哪儿”变成一条能解释、会改主意的决策链。

周末活动真正难的，不是生成更多选项，而是在排队、关店、意见冲突出现时，仍给出一条能执行、能追溯、敢于承认不确定的方案。

BJ-Pal is a reproducible planning agent for short activities in Beijing. It grounds plans in typed evidence, fails closed on missing constraints, persists recoverable jobs, bounds LLM execution, and exposes verifiable operational evidence.

01 / Problem framing

用户不是缺一个行程表，
而是不想独自承担选错的责任。

BJ-Pal 把群体短时活动看成“带现实约束的共同决策”：选项必须合适，也必须在风险出现后及时说明原因、给出替代，并让所有人看得懂。

“你只要出现，
剩下的我安排。”
产品命题 · 情绪基础设施，而非路线生成器
  1. 01

    责任要可见

    每一步保留 decision、confidence 与 fallback，让系统为什么这样安排可以被追问。

  2. 02

    吐槽比星级更接近现场

    把排队、停车、拥挤、真实价格等 UGC soft signals 和 POI facts 一起纳入候选排序。

  3. 03

    方案必须允许反悔

    AvailabilityProbe 发现风险后，只替换失败步骤，不把整条计划推倒重来。

  4. 04

    群体共识不是平均分

    用偏好收敛、否决与 Kemeny+Borda 投票保留分歧，再产出可转发结果。

  5. 05

    重要场合需要人做决定

    高风险关键词触发筛选模式；此时系统退回为解释充分的工具，而不是替人拍板的代理。

证据边界：以上信号来自 100 条 AI 用户访谈，是需求先验与假设生成，不等同于真实用户研究。仓库把真实访谈和可用性验证列为后续工作。

02 / Product systems work

不是在界面上接一个模型，
而是让产品判断进入系统。

Jason Xun 担任项目负责人，主导产品定位、Agent 编排、评测设计与公开叙事；KeepL 作为共同作者参与项目。归因不代表所有团队产出均由一人完成。

01 / Define

把模糊痛点变成可检查行为

定义责任可见、红旗提示、错误自承认、周末聚焦与重要场合筛选。

02 / Orchestrate

把计划、探测、重排串成控制流

Planner 生成，AvailabilityProbe 回灌风险，Replanner 只替换失败步骤。

03 / Evaluate

把“看起来不错”拆成证据层

contract、独立 verifier、provider observation 与用户结果分轨记录；任何一层都不替代下一层。

03 / System design

从一句话，到一条可回放的决策流水线。

当前主链把入口统一到 PlanRequest → Preflight → PlanningService → PlanResult：缺少高风险约束时在 fan-out 前停止；执行中限制 LLM、数据、工具、重试、实报 token 与安全检查点耗时；同步请求和 durable job 共用同一业务语义。

06 / Limitations

展示系统能力，
也展示判断边界。

可信的案例不是把所有数字写大，而是让读者知道：哪些已经验证，哪些只是原型假设，下一步该验证什么。

  1. 01
    真实用户结果仍为 0

    AI 合成访谈、fixture 与 operator observation 不能替代真实参与者的任务完成、采纳或满意度。

  2. 02
    真实 Provider 只有一个样本

    1464 实报 token 与约 28.9 秒只描述该次 DeepSeek observation，不能估计成功率、延迟分布或金额成本。

  3. 03
    评测大多是 hand-authored synthetic

    独立 verifier 能证明契约与工件一致，不能证明 case 分布代表真实北京出行需求。

  4. 04
    可拉取镜像不是公开 API

    GHCR 镜像可匿名读取并完成一次 hardened smoke，但线上入口仍是静态 Pages；没有 TLS 反向代理、多实例服务、真实负载或持续 SLO 证据。

  5. 05
    外部服务仍停在授权边界

    预订只允许沙箱；POI/路线不是合法 live provider；也未实现服务端 KMS、凭证轮换、跨实例预算或 billing reconciliation。
