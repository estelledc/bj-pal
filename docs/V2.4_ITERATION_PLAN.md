### 详细任务拆解

**D3 分层行为评测**（参考 video-eval-agent 三阶段防火墙）

```
evals/behavioral/
  L1_anchor/        # 5 case，每 commit 跑，30s
    - red_flags_visible.yaml
    - apology_after_2_fails.yaml
    - weekend_afternoon_focus.yaml
    - vetoer_triggers_reroute.yaml
    - tool_for_important_event.yaml
  L2_integration/   # 50 场景，每周跑，5min
    - time_buckets/ (周六下午 / 周五晚 / 周日上午)
    - group_dynamics/ (4 人 / 3 人 + 1 沉默 / 隐性领导)
    - multimodal/ (截图 / 公众号 / 语音)
  L3_full/          # 500 场景，每 release 跑，30min
    - 复用 docs/USER_RESEARCH_FINDINGS.md 的 100 AI 用户 × 5 强信号
```

度量产物：每次跑后写入 `evals/results/<commit-sha>.json`，含 5 强信号通过率、F1 gap、退化告警。

**D5 群偏好收敛器**

新增 `src/agents/group_dynamics.py`：

| 成员模式 | 检测信号 | 处理策略 |
|---|---|---|
| 反复横跳 | 否决 ≥ 2 次 | 退出群聊转单聊澄清 |
| 沉默 | broadcast 5min 无响应 | 主动 @ + 给 2 选项 |
| 隐性领导 | 第 1 句定调（vision/keyword 检测） | weight ×1.5 注入 group_harmony |

接入点：`src/agents/group_harmony.py` 加 `weight` 输入参数；broadcast 历史走 SQLite 新表 `group_member_profile`。

**D1 履约 trace 内核**

只做内核，不做 SLA 外壳：

- `src/agents/plan_tracer.py`（新）— 每步 plan 落 SQLite 时记录 `(step_id, decision, confidence, fallback_action)`
- `src/ui/trust_panel.py`（新）— 侧栏可展开，显示"AI 这步 70% 确定，因为 UGC 厚度只 5 条"
- 置信度校准：跑完 100 case 后用 ECE（Expected Calibration Error）量化"AI 说 70% 确定时是不是真的 70% 对"

## 迭代日志

### Round 1 — Baseline（前一轮对话给出）

7 个方向：D1 责任契约 / D2 多模态首屏 / D3 评测自动化 / D4 长尾扩展 / D5 群收敛器 / D6 stateful / D7 可视化。

v2.4 推荐：D1 + D2 + D3，6d。

**问题**：方向是直觉拍的，没经过批评。

---

### Round 2 — Critique: META POSITION 是否真信号？

README 说"100 条 AI 用户访谈"得到 5 强信号。最强是"选错的责任 (4/5)"和"选错容忍度 = 2 次 (5/5)"。

**Gap**：这是 AI 模拟用户访谈，不是真实用户。AI 用户倾向反映"理论上应该说的话"，可能放大情绪需求。

**Revision**：D3 评测自动化应该首先做 "5 强信号是否在真实使用中也是 top-2"。优先级：在 D1 之前先做 D3，否则 D1 是建在 AI 假设上。

新顺序：**D3 → D1 → D2**

---

### Round 3 — Critique: D1 责任契约的可执行性

"AI 替我扛了选错的责任"听起来美，但 mock 数据下怎么"扛"？真赔钱不可能（demo 阶段），话术补偿（"下家加 10min"）只是话术。

**Gap**：责任契约如果只是 UI 卡片会被看穿。要做实需要可验证的"承诺-兑现"链条。

**Revision**：D1 改两层：

- **内核**：plan trace 可视化（每步置信度 + 失败回退动作）— 可验证
- **外壳**：信任档案（跨 session 累积"AI 履约率"）— 持续积累

不做空头 SLA。改名：D1 "责任契约" → D1 "履约可验证 trace"

---

### Round 4 — Critique: D3 评测自动化的成本/收益

100 个虚拟用户 × 5 强信号 × 每 commit 跑 = LongCat tokens 成本不小。

**Gap**：全量跑成本高，子集跑信号弱。

**Revision**：分层评测（L1/L2/L3）

- L1 anchor（每 commit）：5 case，~30s
- L2 集成（每周）：50 场景，~5min
- L3 全量（每 release）：500 场景，~30min

参考 video-eval-agent 的 gstack 三阶段防火墙（intern-journal memory `project_video_eval_agent_constraints.md`）。

---

### Round 5 — Critique: 用户分层缺失

D1-D7 都假设了一种用户。META POSITION 暗示两层：

- **个人决策困难型**（"减脂还是好吃"）→ 需要偏好镜子 + 多模态输入
- **群体决策困难型**（"4 个朋友都说随便"）→ 需要群发投票 + 否决重 reroute

**Gap**：没有分用户做优先级排序。

**Revision**：v2.4 范围内聚焦**群体决策困难型**（这是 META POSITION 最强场景，"周六下午 4 个朋友"是命题字面）。

新 v2.4 主线：**D3 + D1 + D5**（不是原来的 D3+D1+D2）

D2 多模态首屏挪到 v2.5（对个人型用户增益更大）。

---

### Round 6 — Critique: 差异化抗 LLM 进步

哪几条会被 LLM 通用能力侵蚀？

| 方向 | LLM 进步影响 | 评级 |
|---|---|---|
| D1 trace | 这是产品机制不是模型能力 | 抗侵蚀 |
| D2 多模态 | vision 能力会被普及 | 会被侵蚀 |
| D3 评测 | 方法论可复制但具体 fixture 是私有 | 半抗 |
| D5 群收敛 | 群体动力学需要数据，LLM 补不上 | 抗侵蚀 |

**Revision**：投资优先级偏向"抗侵蚀"项。最终 v2.4：**D3 + D1 + D5**。D2 加 priority warning："窗口期 6 个月，再不下沉首屏就被 GPT-5 普及抹平"。

---

### Round 7 — Critique: 缺度量目标

每个方向的"用户可见冲击"是定性的。Boil the lake 应该有定量目标。

**Gap**：没有 success metric。

**Revision**：每个方向加 numeric target：

| 方向 | v2.4 度量目标 |
|---|---|
| D3 行为评测 | 5 强信号通过率建立基线 ≥ 60% |
| D1 履约 trace | plan trace 完整覆盖率 100%，置信度校准 ECE ≤ 0.15 |
| D5 群收敛器 | 4 人群 reroute 收敛轮次中位数 ≤ 2 轮 |

---

### Round 8 — Critique: 风险评估缺位

每个方向的失败模式没说。

**Risks & Mitigations**：

| 风险 | 影响 | 缓解 |
|---|---|---|
| LongCat API 限流/改协议 | 阻塞所有 LLM 链路 | `llm_client.py` 多模型抽象（已有），确认 anthropic fallback 通 |
| UGC 数据合规（Class A 公开汇总 479 条） | 版权风险 | 仅做 aspect 切片不存原文；标注来源 5 类透明 |
| Mock 真实化的边界 | 评委追问"真预订怎么办" | 预设答案（"真预订接美团商家 SDK，5 天工"），写进 QA_PREP |
| D3 评测污染（fixture 反向训练 LLM） | 评测信号失效 | 借鉴 gstack 三阶段防火墙：fixture 与 production prompt 分库 |

---

### Round 9 — Critique: Jason 是零基础学习者

global memory `user_background.md` 写明 Jason 编程零基础但擅长 CC 辅助学习。v2.4 工作量 6d 是 AI 视角。

Jason 实际：

- 跟读 + 学习每条 commit 的"why"
- 单日有效编码窗口 4-6h
- 调试卡点问 CC，但不能跳过排查直接给答案（教学节奏 feedback）

**Revision**：6d AI 估算 × 1.5-2.0 系数 → 真实工期 9-12d。

如果窗口紧，砍到 **D3 + D5**（D1 推 v2.5），3-5d 可达。

---

### Round 10 — 收敛: 最小可演示切片

第一性推导：评委 5min demo 看到的可信故事是什么？

**最小可信故事**：群体决策困难（4 个朋友周六下午）→ AI 主动收敛（识别否决/沉默/领导模式）→ 履约 trace 看得到（每步置信度 + fallback）。

满足这个故事的最小集 = **D3 评测 + D5 群收敛 + D1 trace 内核**。

D1 SLA 外壳砍掉（mock 阶段说不清"赔付"），D2/D4/D6/D7 全推后。

最终工期：5d AI / 8d Jason。
